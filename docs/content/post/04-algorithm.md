expls の「色」はどうやって決まっているのでしょうか。
ここでは仕組みを、できるだけやさしく説明します。

色は次の **2つの軸** で決まります。

1. **拡張子** → 色あい（色相）を決める
2. **更新の新しさ** → 色の濃さ（鮮やかさ・明るさ）を決める

順番に見ていきましょう。

## 軸1：拡張子で「色あい」を決める

ファイルの拡張子（`.rs`, `.py`, `.png` など）ごとに、
**色相（hue）** という「色あいを表す角度」を割り当てています。

色相は 0〜360 の数字で、円のように一周します。

- 0 / 360 付近 … 赤
- 120 付近 … 緑
- 240 付近 … 青

たとえば expls では、おおよそ次のように割り当てています。

| 拡張子 | 色あい（色相） | だいたいの色 |
|---|---|---|
| `.rs`（Rust） | 25 | オレンジ |
| `.py`（Python） | 215 | 青 |
| `.js` | 50 | 黄 |
| `.md` | 175 | シアン |
| `.png` などの画像 | 325 | ピンク |
| ディレクトリ | 220 | 青 |

同じ言語・同じ仲間のファイルは似た色になるので、
画面の中で**種類ごとにまとまって見える**わけです。

リストに無い拡張子（未知のファイル）には色あいを割り当てず、**グレー**で表示します。

## 軸2：更新の新しさで「濃さ」を決める

次に、同じ拡張子のファイルを集めて、**更新日時の新しい順に並べます**。
そして並び順から、`0.0`〜`1.0` の数字 `t` を計算します。

- いちばん新しいファイル → `t = 0.0`
- いちばん古いファイル → `t = 1.0`

ファイルが \(n\) 個あるとき、新しい方から数えて \(k\) 番目（0 から数える）の `t` は

$$ t = \frac{k}{\,n - 1\,} $$

で求めます。この `t` を使って、色の **鮮やかさ（彩度）** と **明るさ（明度）** を変えます。

| | いちばん新しい（t=0） | いちばん古い（t=1） |
|---|---|---|
| 彩度（鮮やかさ） | 0.85（高い） | 0.30（低い） |
| 明度（明るさ） | 0.45（暗め） | 0.75（明るい） |

つまり、**新しいほど濃く鮮やか**、**古いほど薄く淡く**なります。
これが「グラデーション」の正体です。

`--reverse` を付けると、この `t` を `1 - t` に置き換えるので、濃淡が逆転します。

## 軸を合体させる：HSL → RGB

ここまでで「色あい（色相）」「鮮やかさ（彩度）」「明るさ（明度）」の
3つがそろいました。この3つの組み合わせ方を **HSL** と呼びます。

ただし、ターミナルに色を出すには **RGB**（赤・緑・青の3つの数値）が必要です。
そこで expls は HSL を RGB に変換し、次のような
**ANSIエスケープシーケンス**で文字に色を付けています。

```text
\x1b[38;2;R;G;Bm  ファイル名  \x1b[0m
```

`R;G;B` の部分が、計算した色の値です。最後の `\x1b[0m` で色をリセットします。

## まとめ：全体の流れ

1. フォルダの中身を読み取る（必要なら隠しファイルやサブフォルダも）
2. 拡張子ごとにファイルをグループ分けする
3. 各グループを更新日時の新しい順に並べ、`t`（0.0〜1.0）を計算する
4. 拡張子 → 色相、`t` → 彩度・明度 を決める
5. HSL から RGB に変換し、色付きで一覧表示する

この5ステップによって、**「種類」と「新しさ」が同時に色で分かる**一覧が
作られています。
